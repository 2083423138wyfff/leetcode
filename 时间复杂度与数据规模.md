# 时间复杂度与数据规模的关系

做算法题时，根据输入规模 `n` 选择合适复杂度的算法，是避免超时的关键。

## 核心直觉

- **时间复杂度**：描述当输入规模 `n` 变大时，程序运行所需操作次数的增长速度。
- **评测机速度**：在线判题系统（如 LeetCode）通常每秒能执行约 **10^7 ~ 10^8** 次简单操作。

## 复杂度与数据规模对照表

| n 的范围 | 允许的复杂度 | 典型做法 |
|---------|------------|---------|
| n ≤ 10 | O(n!), O(2ⁿ) | 暴力枚举、回溯 |
| n ≤ 20 | O(2ⁿ) | 状态压缩 DP |
| n ≤ 100 | O(n³) | 动态规划、Floyd-Warshall |
| n ≤ 1,000 | O(n²) | 双重循环、动态规划 |
| n ≤ 10⁵ | O(n log n) | 排序、二分查找、堆 |
| n ≤ 10⁶ | O(n) | 单次遍历、双指针 |
| n ≤ 10⁹ | O(log n), O(1) | 数学公式、二分答案 |
